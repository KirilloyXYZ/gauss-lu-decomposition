# Сравнение метода Гаусса и LU-разложения

Учебный проект на C++17 по теме решения СЛАУ.  
В проекте реализованы:

- метод Гаусса без выбора ведущего элемента;
- метод Гаусса с частичным выбором главного элемента по столбцу;
- LU-разложение без перестановок;
- прямая и обратная подстановка;
- эксперименты по времени и точности.

Проект сделан без `std::vector` как основного контейнера. Для хранения данных используется свой `DynamicArray`, а на его основе реализованы `Vector` и `Matrix`.

## Постановка задачи

Нужно решить систему линейных алгебраических уравнений

$$
A x = b,
$$

где:

- $A$ — квадратная матрица размера $n \times n$;
- $x$ — вектор неизвестных;
- $b$ — правая часть.

В работе сравниваются:

1. обычный метод Гаусса;
2. метод Гаусса с частичным выбором главного элемента;
3. LU-разложение без перестановок.

Отдельно показывается, почему LU-разложение особенно удобно, когда матрица $A$ фиксирована, а правых частей $b$ несколько.

## Используемые математические формулы

### 1. Метод Гаусса

На шаге $k$ выбирается ведущий элемент $a_{kk}^{(k)}$ и для всех строк $i > k$ вычисляется множитель

$$
m_{ik} = \frac{a_{ik}^{(k)}}{a_{kk}^{(k)}}.
$$

Дальше строка обновляется:

$$
a_{ij}^{(k+1)} = a_{ij}^{(k)} - m_{ik} a_{kj}^{(k)}, \quad j = k+1, \dots, n,
$$

а правая часть:

$$
b_i^{(k+1)} = b_i^{(k)} - m_{ik} b_k^{(k)}.
$$

В результате матрица приводится к верхнетреугольному виду.

### 2. Частичный выбор главного элемента

Для текущего столбца $k$ выбирается строка

$$
p = \arg\max_{i \ge k} |a_{ik}|.
$$

После этого строки $k$ и $p$ меняются местами, и обычный прямой ход продолжается уже с более устойчивым ведущим элементом.

### 3. LU-разложение

Матрица представляется в виде

$$
A = L U,
$$

где:

- $L$ — нижнетреугольная матрица;
- $U$ — верхнетреугольная матрица;
- на диагонали $L$ стоят единицы:

$$
l_{ii} = 1.
$$

Формулы вычисления элементов:

$$
u_{ij} = a_{ij} - \sum_{k=1}^{i-1} l_{ik} u_{kj}, \quad j \ge i,
$$

$$
l_{ij} = \frac{a_{ij} - \sum_{k=1}^{j-1} l_{ik} u_{kj}}{u_{jj}}, \quad i > j.
$$

Если на диагонали $U$ появляется слишком маленький элемент, разложение без перестановок считается невозможным.

### 4. Прямая подстановка

После LU-разложения сначала решается система

$$
L y = b.
$$

Компоненты $y_i$ вычисляются сверху вниз:

$$
y_i = \frac{b_i - \sum_{j=1}^{i-1} l_{ij} y_j}{l_{ii}}.
$$

### 5. Обратная подстановка

Потом решается система

$$
U x = y.
$$

Компоненты $x_i$ вычисляются снизу вверх:

$$
x_i = \frac{y_i - \sum_{j=i+1}^{n} u_{ij} x_j}{u_{ii}}.
$$

### 6. Евклидова норма

Для вектора $v = (v_1, \dots, v_n)^T$ используется норма

$$
\|v\|_2 = \sqrt{\sum_{i=1}^{n} v_i^2}.
$$

### 7. Невязка

Для найденного приближённого решения $\tilde{x}$ вычисляется вектор невязки

$$
r = A \tilde{x} - b.
$$

Норма невязки:

$$
\|r\|_2 = \|A \tilde{x} - b\|_2.
$$

### 8. Относительная ошибка

Если известно точное решение $x$, то относительная ошибка считается так:

$$
\frac{\|\tilde{x} - x\|_2}{\|x\|_2}.
$$

### 9. Матрица Гильберта

В третьем эксперименте используется матрица Гильберта:

$$
H_{ij} = \frac{1}{i + j - 1}, \quad i, j = 1, \dots, n.
$$

Это классический пример плохо обусловленной матрицы, на которой хорошо видны численные ошибки.

## Что делает программа

Доступные команды:

```bash
./program selfcheck
./program single
./program multi
./program hilbert
./program all
```

Назначение команд:

- `selfcheck` — маленькая самопроверка на фиксированной системе;
- `single` — сравнение времени для одной системы;
- `multi` — сравнение времени для нескольких векторов `b` при одной матрице;
- `hilbert` — сравнение точности на матрицах Гильберта;
- `all` — запуск всех экспериментов подряд.

## Как собрать

Нужны `g++` и `make`.

```bash
make
```

После сборки появляется исполняемый файл `program`.

## Схема экспериментов

### Эксперимент 1. Одна система

Для размеров

$$
n \in \{100, 200, 500, 1000\}
$$

генерируются:

- случайная матрица $A$;
- случайный вектор $b$.

Значения генерируются из равномерного распределения

$$
U[-1, 1].
$$

Измеряется время:

- метода Гаусса без выбора;
- метода Гаусса с выбором;
- LU-разложения;
- решения через готовые $L$ и $U$;
- полного LU-варианта:

$$
T_{LU} = T_{decompose} + T_{solve}.
$$

### Эксперимент 2. Несколько векторов `b`

Фиксируется одна матрица размера

$$
n = 500.
$$

Дальше рассматриваются значения

$$
k \in \{1, 10, 100\},
$$

где $k$ — число разных векторов $b$.

Сравниваются два подхода:

1. Гаусс с partial pivoting, который для каждого нового $b$ решает систему заново.
2. LU-подход, где разложение строится один раз, а потом для каждого нового $b$ выполняются только подстановки.

### Эксперимент 3. Точность на матрицах Гильберта

Для размеров

$$
n \in \{5, 10, 15\}
$$

строится матрица Гильберта $H$.

Точное решение выбирается таким:

$$
x = (1, 1, \dots, 1)^T.
$$

Потом считается правая часть:

$$
b = Hx.
$$

После этого система решается тремя методами, и для каждого считаются:

- относительная ошибка;
- норма невязки.

## Краткие выводы

По полученным результатам видно следующее:

- для одной системы LU не всегда выгоднее, потому что само разложение достаточно дорогое;
- при нескольких векторах `b` LU быстро начинает выигрывать, так как разложение строится один раз;
- метод Гаусса с частичным выбором ведущего элемента устойчивее, чем вариант без выбора;
- на матрицах Гильберта у всех методов растёт ошибка, особенно при увеличении размера;
- маленькая невязка не гарантирует маленькую ошибку решения на плохо обусловленных матрицах.
