# Решение СЛАУ методом Гаусса и методом LU-разложения

Лабораторная работа по информатике на C++.

## Описание проекта

В проекте реализовано решение систем линейных алгебраических уравнений следующими методами:

- метод Гаусса без выбора ведущего элемента;
- метод Гаусса с выбором главного элемента по столбцу;
- LU-разложение матрицы без перестановок;
- прямая подстановка;
- обратная подстановка.

Также в программе реализованы:

- генерация случайных матриц и правых частей;
- генерация матрицы Гильберта;
- вычисление невязки;
- вычисление относительной погрешности;
- замер времени работы алгоритмов;
- запуск трёх экспериментов, предусмотренных лабораторной работой.

---

## Постановка задачи

Требуется реализовать набор функций для решения систем линейных алгебраических уравнений (СЛАУ) методом Гаусса и методом LU-разложения.

Необходимо:

- сравнить время работы алгоритмов;
- показать преимущество LU-разложения при решении систем с одной и той же матрицей, но разными правыми частями;
- проверить точность методов на плохо обусловленных матрицах Гильберта.

---

## Реализованные алгоритмы

### 1. Метод Гаусса без выбора ведущего элемента

Алгоритм состоит из двух частей:

- прямой ход — приведение матрицы к верхнетреугольному виду;
- обратный ход — нахождение решения снизу вверх.

### 2. Метод Гаусса с выбором главного элемента по столбцу

На каждом шаге выбирается строка с максимальным по модулю элементом в текущем столбце, после чего строки меняются местами. Это повышает численную устойчивость метода.

### 3. LU-разложение

Матрица представляется в виде:

A = L * U

где:

- `L` — нижнетреугольная матрица;
- `U` — верхнетреугольная матрица;
- на диагонали `L` стоят единицы.

После разложения решение системы сводится к двум этапам:

1. решению системы `Ly = b` методом прямой подстановки;
2. решению системы `Ux = y` методом обратной подстановки.

---

## Метрики качества решения

В программе вычисляются:

### Невязка

||Ax - b||

### Относительная погрешность

||x_approx - x_exact|| / ||x_exact||

Для вычисления нормы используется евклидова норма.

## Эксперименты

### Эксперимент 1. Сравнение времени решения одной системы

Для размеров:

- `n = 100`
- `n = 200`
- `n = 500`
- `n = 1000`

генерируются:

- случайная матрица `A`;
- случайная правая часть `b`.

Измеряется время:

- метода Гаусса без выбора ведущего элемента;
- метода Гаусса с выбором главного элемента;
- LU-разложения;
- решения через уже готовые матрицы `L` и `U`.

---

### Эксперимент 2. Экономия времени при нескольких правых частях

Фиксируется одна матрица размера:

- `n = 500`

Далее рассматриваются случаи:

- `k = 1`
- `k = 10`
- `k = 100`

где `k` — число различных правых частей.

Сравниваются:

- общее время решения всех систем методом Гаусса с выбором ведущего элемента;
- общее время решения всех систем через LU-разложение:
  - одно разложение,
  - затем решение треугольных систем для каждой правой части.

---

### Эксперимент 3. Проверка точности на матрицах Гильберта

Для размеров:

- `n = 5`
- `n = 10`
- `n = 15`

строится матрица Гильберта:

H[i][j] = 1 / (i + j - 1)

В качестве точного решения берётся вектор:

x = (1, 1, ..., 1)^T

После этого вычисляется правая часть:

b = Hx

Далее система решается всеми реализованными методами, и для каждого решения вычисляются:

- относительная погрешность;
- невязка.

---

## Что делает программа

После запуска программа автоматически выполняет три эксперимента подряд:

1. сравнение времени решения одной системы;
2. сравнение времени при нескольких правых частях;
3. сравнение точности на матрицах Гильберта.

Результаты выводятся в консоль.

## Структура проекта

- `main.cpp` — точка входа, запуск всех экспериментов;
- `gauss.h`, `gauss.cpp` — метод Гаусса без выбора ведущего элемента и метод Гаусса с выбором главного элемента по столбцу;
- `lu.h`, `lu.cpp` — LU-разложение без перестановок, прямая и обратная подстановка;
- `generators.h`, `generators.cpp` — генерация случайных матриц и векторов, генерация матрицы Гильберта, умножение матрицы на вектор, создание вектора из единиц;
- `metrics.h`, `metrics.cpp` — вычисление нормы, невязки и относительной погрешности;
- `benchmark.h`, `benchmark.cpp` — функции замера времени выполнения алгоритмов;
- `experiments.h`, `experiments.cpp` — реализация трёх экспериментов;
- `README.md` — описание проекта, инструкция по сборке и запуску.
