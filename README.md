# Решение СЛАУ методом Гаусса и методом LU-разложения

Лабораторная работа по информатике на C++.

## Описание проекта

В проекте реализовано решение систем линейных алгебраических уравнений (СЛАУ) следующими методами:

- метод Гаусса без выбора ведущего элемента;
- метод Гаусса с выбором главного элемента по столбцу;
- LU-разложение матрицы без перестановок;
- прямая подстановка;
- обратная подстановка.

Также в программе реализованы:

- генерация случайных матриц и правых частей;
- генерация матрицы Гильберта;
- вычисление невязки;
- вычисление относительной погрешности;
- замер времени работы алгоритмов;
- запуск трёх вычислительных экспериментов.

---

## Постановка задачи

Требуется реализовать набор функций для решения систем линейных алгебраических уравнений вида

$$
A x = b,
$$

где:
- $A$ — квадратная матрица коэффициентов,
- $x$ — вектор неизвестных,
- $b$ — вектор правой части.

Необходимо:

- сравнить время работы алгоритмов;
- показать преимущество LU-разложения при решении систем с одной и той же матрицей, но разными правыми частями;
- проверить точность методов на плохо обусловленных матрицах Гильберта.

---

## Реализованные алгоритмы

## 1. Метод Гаусса без выбора ведущего элемента

Метод состоит из двух этапов:

### Прямой ход

На шаге $k$ для каждой строки $i > k$ вычисляется коэффициент:

$$
\text{factor} = \frac{A_{ik}}{A_{kk}}.
$$

После этого выполняется зануление элементов ниже диагонали:

$$
A_{ij} := A_{ij} - \text{factor} \cdot A_{kj},
$$

$$
b_i := b_i - \text{factor} \cdot b_k.
$$

### Обратный ход

После приведения матрицы к верхнетреугольному виду решение находится по формуле:

$$
x_i = \frac{b_i - \sum_{j=i+1}^{n-1} A_{ij} x_j}{A_{ii}}.
$$

---

## 2. Метод Гаусса с выбором главного элемента по столбцу

На каждом шаге выбирается строка с максимальным по модулю элементом в текущем столбце:

$$
|A_{pk}| = \max_{i \ge k} |A_{ik}|.
$$

После этого строки меняются местами, и выполняется обычный шаг метода Гаусса.  
Такой подход повышает численную устойчивость метода.

---

## 3. LU-разложение

Матрица представляется в виде:

$$
A = L U,
$$

где:
- $L$ — нижнетреугольная матрица;
- $U$ — верхнетреугольная матрица;
- на диагонали $L$ стоят единицы.

### Формулы вычисления элементов LU-разложения

Элементы матрицы $U$ вычисляются по формуле:

$$
U_{ik} = A_{ik} - \sum_{j=0}^{i-1} L_{ij} U_{jk}, \quad k \ge i.
$$

Элементы матрицы $L$ вычисляются по формуле:

$$
L_{ki} = \frac{A_{ki} - \sum_{j=0}^{i-1} L_{kj} U_{ji}}{U_{ii}}, \quad k > i.
$$

---

## 4. Прямая подстановка

После LU-разложения сначала решается система

$$
L y = b.
$$

Элементы вектора $y$ вычисляются по формуле:

$$
y_i = \frac{b_i - \sum_{j=0}^{i-1} L_{ij} y_j}{L_{ii}}.
$$

---

## 5. Обратная подстановка

Затем решается система

$$
U x = y.
$$

Элементы решения вычисляются по формуле:

$$
x_i = \frac{y_i - \sum_{j=i+1}^{n-1} U_{ij} x_j}{U_{ii}}.
$$

---

## Метрики качества решения

### Евклидова норма вектора

Для вектора $v = (v_1, v_2, \dots, v_n)$ используется евклидова норма:

$$
\|v\| = \sqrt{\sum_{i=1}^{n} v_i^2}.
$$

### Невязка

Невязка решения определяется как:

$$
r = A x - b,
$$

а её величина вычисляется по формуле:

$$
\|A x - b\|.
$$

### Относительная погрешность

Если известно точное решение $x_{\text{exact}}$ и найдено приближённое решение $x_{\text{approx}}$, то относительная погрешность равна:

$$
\frac{\|x_{\text{approx}} - x_{\text{exact}}\|}{\|x_{\text{exact}}\|}.
$$

---

## Матрица Гильберта

Для проверки точности используется матрица Гильберта, элементы которой задаются формулой:

$$
H_{ij} = \frac{1}{i + j - 1}.
$$

В реализации на C++ индексация начинается с нуля, поэтому в коде используется эквивалентная формула:

$$
H_{ij} = \frac{1}{i + j + 1}.
$$

## Эксперимент 1. Сравнение времени решения одной системы

Для размеров:
- $n = 100$,
- $n = 200$,
- $n = 500$,
- $n = 1000$

генерируются:
- случайная матрица $A$;
- случайная правая часть $b$.

Измеряется время:
- метода Гаусса без выбора ведущего элемента;
- метода Гаусса с выбором главного элемента;
- LU-разложения;
- решения через уже готовые матрицы $L$ и $U$.

При решении одной системы методы Гаусса и LU имеют близкий порядок сложности.  
LU-разложение требует отдельного этапа построения матриц $L$ и $U$, поэтому не даёт заметного выигрыша по времени для одной правой части.

## Эксперимент 2. Экономия времени при нескольких правых частях

Фиксируется одна матрица размера:
- $n = 500$

Рассматриваются случаи:
- $k = 1$,
- $k = 10$,
- $k = 100$,

где $k$ — число различных правых частей.

Сравниваются:
- общее время решения всех систем методом Гаусса с выбором главного элемента;
- общее время решения всех систем через LU-разложение:
  - одно разложение,
  - затем решение треугольных систем для каждой правой части.

### Результаты одного запуска

| k | GaussPartialPivotTotal, c | LUTotal, c |
|---:|---:|---:|
| 1   | 0.210000 | 0.230000 |
| 10  | 2.170000 | 0.240000 |
| 100 | 21.420000 | 0.310000 |

### Вывод по эксперименту 2

При увеличении числа правых частей LU-разложение становится значительно выгоднее.  
Это объясняется тем, что разложение матрицы выполняется один раз, а затем для каждой новой правой части решаются только две треугольные системы.

## Эксперимент 3. Проверка точности на матрицах Гильберта

Для размеров:
- $n = 5$,
- $n = 10$,
- $n = 15$

строится матрица Гильберта $H$.

В качестве точного решения берётся вектор:

$$
x = (1,1,\dots,1)^T.
$$

Далее вычисляется правая часть:

$$
b = Hx.
$$

После этого система решается всеми реализованными методами, и для каждого решения вычисляются:
- относительная погрешность;
- невязка.

| LU | 1.340568e-11 | 5.087681e-16 |

#### n = 10

### Вывод по эксперименту 3

На матрицах Гильберта с ростом размера резко ухудшается точность решения, поскольку такие матрицы плохо обусловлены.  
Метод Гаусса с выбором главного элемента показывает более устойчивое поведение, чем метод без выбора.  
При этом невязка остаётся очень маленькой, даже если относительная погрешность уже велика.

## Что делает программа

После запуска программа автоматически выполняет три эксперимента подряд:

1. сравнение времени решения одной системы;
2. сравнение времени при нескольких правых частях;
3. сравнение точности на матрицах Гильберта.

Результаты выводятся в консоль.
